# 生成AI時代に「社内SE求人」が増える本当の理由

## 結論（一言で）
生成AIは「外注で済む仕事」を減らし、「社内に置かないと回らない仕事」を増やしている。

## なぜ今これを知るべきか（why_now）
日本企業の生成AI導入率は2025年に57.7%へ到達し、前年から+12.9ポイントの急伸（NRI 2025）。同時にIT求人倍率は10.4倍（レバテック 2025/12）、生成AI関連求人は前年比410%。「社内SE＝守りの情シス」だった時代は終わり、AI実装・ガバナンス・内製の旗振り役を担える人材が、事業会社側の高待遇枠で取り合いになっている。今が転職市場としての非対称性が最大化しているタイミング。

## ブロック設計（図解の各パーツ）

### Block 1: マクロ背景 ―「生成AIが日本企業に定着した1年」
- ポイント: 国内導入率は1年で12.9pt急伸。上場・大企業を中心に「PoC終了→運用フェーズ」へ移行
- 詳細:
  - 生成AI導入済み企業：2023年度33.8% → 2024年度44.8% → **2025年度57.7%**（NRI「IT活用実態調査2025」）
  - 業種別では情報通信業64.4%、金融・保険業42.4%（NRI）
  - 大企業役員層の利用率52.3%、係長〜部長層は54.3%（INTAGE 2025/11）
  - 法人での生成AI利用社数：2024年末32.7万社 → 2026年予測50.1万社（ICT総研）
- 出典: NRI / INTAGE / ICT総研 / 総務省情報通信白書

### Block 2: 構造変化① ―「PoC→運用」で情シスに業務が"落ちてくる"
- ポイント: 社内ChatGPT・RAG・Copilot は導入後の運用主体が情シス／社内SEに集中
- 詳細:
  - PwC調査：日本企業の56%が「社内で生成AI活用中／社外提供中」（前回比+13pt）
  - 一方で「期待を大きく上回る効果を得た」企業は10%（米国45%）。**運用・定着フェーズが課題**＝そこを担う人材＝社内SE
  - 業務領域の具体例：社内RAG構築、Azure OpenAI / GitHub Copilot Enterprise の権限・コスト管理、社内向けプロンプト・テンプレート整備、ヘルプデスクのAIチャット化
  - 日立の事例：GitHub Copilot Enterprise＋社内開発フレームワークで一部業務ロジック開発のコード生成率99%
- 読者へのインプリケーション: 「AIをツール導入できる」より「AIを業務に組み込み・運用しきれる」人が圧倒的に不足
- 出典: PwC「生成AI実態調査2025春」/ AINOW / JBpress

### Block 3: 構造変化② ―「シャドーAI／情報漏えい対策」のためのガバナンス需要
- ポイント: 「禁止」から「管理付き許可」へ。誰かが社内ルールと監視基盤を作らないといけない
- 詳細:
  - NRI調査：生成AI活用課題として企業の70.3%が「リテラシー・スキル不足」、48.5%が「リスク把握・管理が難しい」
  - PwC：生成AIを「コンプライアンス・企業文化への脅威」と捉える企業が44%（前回比+23pt）
  - FISC（金融）が2025年に生成AI利用ガイドラインを策定。規制業種ほど社内SE側の整備負荷が増加
  - レバテック：セキュリティ人材の求人倍率は**42.6倍**、直近3年で求人数2.5倍に拡大
- 読者へのインプリケーション: セキュリティ＋AIガバナンス（DLP、IRM、ログ監査、利用ガイドライン）が分かる社内SEが希少枠
- 出典: NRI / PwC / レバテック（マイナビTechPlus）/ 第一生命経済研究所

### Block 4: 構造変化③ ―「ベンダー丸投げ」から内製・伴走型への揺り戻し
- ポイント: 生成AIで開発工数が下がり、事業会社が「自分で作る／使いこなす」方向に振れた
- 詳細:
  - IPA「DX動向2025」：上流の「企画」内製化は65%まで進む一方、「開発」は17%・「テスト」は23%にとどまる ⇒ 開発フェーズの内製化余地が大きく、社内SE採用の主戦場
  - レバテック調査：**生成AI関連の正社員/フリーランス求人は前年比410%増**（2024年6月時点）。事業会社側の発注比率が上昇
  - ノーコード／ローコード導入率は2025年に51.0%（NRI）。市民開発の旗振り役＝社内SE
  - dodaの2025年9〜11月「入社決定職種」では「社内情報システム（社内SE）」が最多カテゴリの一つ（25〜37%レンジ）
- 読者へのインプリケーション: SIer/SESで開発経験を積んだ人が、事業会社側の内製リーダーとしてプレミアム年収で迎えられる構造
- 出典: IPA / レバテック / NRI / doda

### Block 5: マクロ供給 ―「IT人材79万人不足」の本丸はAI実装人材
- ポイント: 数の不足は不可逆。社内SEの売り手市場は2030年まで続く前提でよい
- 詳細:
  - 経産省試算：2030年までに最大**79万人**のIT人材不足。従来型IT人材は約10万人余剰、一方で先端IT人材が大幅不足（※経産省「IT人材需給に関する調査」）
  - 全業種の有効求人倍率1.18倍に対し、ITエンジニア（レバテック調査）は**10.4倍**（2025/12）
  - dodaの中途求人倍率も2025年11月に2.70倍（前月比+0.20pt）と上昇
  - 情報処理・通信技術者の有効求人倍率1.43倍（2025/11、厚労省ベース／日経）
- 出典: 経産省 / 日経 / レバテック / doda

## キーメッセージ（図解の周辺テキスト用）

- 求人倍率10.4倍／セキュリティは42.6倍

## 読者アクション（CTA：社内SE志望者が今やるべき3step）
1. **AI実装ポートフォリオを作る**：Azure OpenAI / Copilot / 簡易RAG をローカル＋クラウドで動かし、GitHub or Notion で公開。「ツール紹介」ではなく「社内業務に落とす」視点で1つ完成させる。
2. **セキュリティ＋ガバナンスの語彙を仕入れる**：FISC、NIST AI RMF、JISQ27001、DLP、IRM、社内利用ガイドラインの雛形を1本書ける状態に。求人票の頻出キーワードに対する自分の語彙を増やすだけで通過率が変わる。
3. **「事業会社×内製」軸で求人を見る**：SIer/SESは"通過点"扱いに切り替え、レバテックダイレクト・doda・Green の "社内SE / 情シス / DX推進" の中でも「内製・生成AI・PMO」キーワードがある求人を10件ストック。職務経歴書を「業務理解 × 内製化推進」の物語にリライト。

## 落とし穴・注意点
- 「社内SE＝楽」という古い印象は危険。生成AI時代の社内SEは**経営直結の意思決定**（コスト・リスク・全社展開）を求められ、待ちの運用保守だけのポジションは縮小傾向（IPA DX動向2025の問題提起）。
- 求人倍率10.4倍は「IT全体」の数字。**従来型運用保守のみの人材は供給過剰側**になりつつあり、需給ギャップは"AI/クラウド/セキュリティに寄せた人"だけが享受する。
- 生成AI関連求人の410%増は2024年6月時点のレバテック数値であり、母数が小さい段階の伸び率。**水準の高さより方向性の指標として読む**こと。
- PwC調査の「効果を実感した企業10%」は裏返すと、運用フェーズで詰まっている企業が多い＝社内SE側の責任範囲が今後さらに膨らむ警戒シグナルでもある。

## 研究メモ（diagrammer/reviewer 用補足）

### 採用したストーリー構造
「需要が増えている」だけでなく「なぜ社内に置くしかないのか」を3つの構造変化（運用フェーズ移行 / ガバナンス / 内製化）で示し、それを支えるマクロ数値（人材不足79万人・求人倍率10.4倍）で締めるピラミッド構造。1枚図解では「結論→構造3つ→KPI→アクション」のZ字導線が読みやすい。

### 数値の信頼度ランキング
- 高: 経産省（79万人）、総務省情報通信白書、IPA DX動向、NRI、PwC、ICT総研、INTAGE、doda公式、レバテック公式
- 中: 社内SEナビ（媒体だが転職エージェント運営、年収中央値の数値根拠は推計込み）
- 不採用: 個人ブログ、断片的な転載記事

### 数値の整合性チェック
- 生成AI導入率は調査により幅がある：JUAS 41.2%（2025）／NRI 57.7%（2025）／総務省（白書） 49.7%（2024年度）。いずれも"CIO層が回答 vs ビジネスパーソン全体"など対象が異なる。図解では「最も新しく、CIO対象で意思決定密度が高い」NRI 57.7%を主軸に据え、業種別はNRI、職位別はINTAGEを併用した。
- 求人倍率もレバテック（IT特化、転職サイト系）10.4倍と、doda（全職種、転職市場全体）2.70倍は数値が大きく違う。図解では「IT人材＝レバテック」「全体相対比較＝doda」と分けて示す。

### 未確認・要追加調査
- 「社内SE職種」単独での求人倍率推移の長期データは公開ソースが薄い。doda公式 ITエンジニアレポートでは"社内SE登録者比率"はあるが"社内SE求人倍率"は非開示。
- IPA「DX動向2025」のPDF本体未読。AI人材の質的不足比率（％）を取り切れていない。次回深掘り時にPDF DLして追加すべき。
- "ベンダー丸投げから内製"の定量データは IPA DX白書経由の "開発工程内製化17%" のみで、年次推移までは未取得。

### 過去ナレッジへの追加候補（agent-memory）
- 信頼ソース帯：生成AI導入率はNRI/JUAS/INTAGE/総務省で年1回更新。年初〜春に最新版が出る傾向。
- 求人倍率の語り分け：レバテック＝IT特化高倍率、doda＝全体俯瞰、リクルートエージェント＝大手寄り。図解では2軸併記が誤解を生まない。
