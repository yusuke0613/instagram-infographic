# 社内SE書類選考を通す職務経歴書7つのチェックポイント

## 既存MDとの差別化ポイント
- `2026-05-09_shanai-se-interview-5questions-templates.md` は**面接段階**のテンプレ。本MDは**書類段階**に特化。
- `2026-05-09_shanai-se-job-description-mine-words.md` は**求人票を読む側**の地雷ワード辞書。本MDは**書類を出す側**のチェックリスト。
- `2026-05-09_ses-3nen-shanai-se-7realities.md` は**転職全体の現実**。本MDは**通過しない原因の分解**にフォーカス。

## 結論（一言で）
通過率を分けるのは経験の量ではなく「事業会社の言葉に翻訳できているか」

## なぜ今これを知るべきか（why_now）
2025-2026年は社内SE求人1ポストに100倍超応募が常態化（doda入社決定職種で社内SEが最多）。さらに2024年8月のジョブ型人事指針公表以降、日立450種・富士通45,000名対象など職務記述書ベースで「スキル要件を明文化」する企業が拡大。書類選考は「経験の有無」ではなく「要件との合致を文章で示せるか」のゲームに変わった。SIer/SES経験5年あっても書き方ひとつで落ちる、を読者は実体験として理解している。

## 図解構成（推奨レイアウト）
- レイアウト案: チェックリスト型7カード（縦並び）+ 上部に「通過率37.3%」のフック数値、下部にCTAブロック
- 視線誘導: 上部の数字インパクト → 7項目スキャン → 下段「添削で通過率は変わる」へ着地

## ブロック設計（図解の各パーツ）

### Block 0（フック）: 経験5年でも落ちる現実
- ポイント: 書類通過率は平均37.3%、約6割は落ちる
- 詳細:
  - マイナビ「転職活動実態調査（2025年）」：書類選考通過率**37.3%**、平均応募**13.6件**、通過は**5.1件**（有効回答467名・2025年7月調査）
  - dodaは個別実感ベースで**約22%**との分析あり。一般水準で**30〜50%**（マイナビ転職）
  - ITは求人倍率10.4倍（レバテック2025/12）でも、社内SEは1ポスト100-200倍応募の人気職
- 出典: マイナビキャリアリサーチLab／doda／レバテック

### Block 1: 担当システムの「規模感」を数字で固定する
- ポイント: 利用者数・PJ人数・予算規模を必ず冒頭に
- 詳細:
  - リクルートエージェント見本「PJ全体150人／利用者グループ10社・約1,800名」のように冒頭で読者に**スケールを固定**させる
  - dodaチェックポイント①「担当システムの業界・規模・環境」が最重要評価軸
  - NG: 「基幹システム改修を担当」 → OK: 「売上1,200億／利用者400名の販売管理刷新（PM2名・PG6名）」
- 出典: リクルートエージェント／doda職務経歴書ガイド／JAC Recruitment

### Block 2: 「対応した／担当した」を全部消す（定量効果に置換）
- ポイント: 動詞を「成果動詞＋数字」に書き換える
- 詳細:
  - 採用担当が見るのは「言われたテストを実施したか／テスト項目自体を作成したか」（マイナビ）
  - 「〇%向上」「〇h削減」「〇件削減」を最低3箇所
  - 例: ヘルプデスク問い合わせ件数を月**200件→90件**（自動化により55%削減）／**月6h**作業時間削減（Power Automate導入）
  - リアル相場: 業務効率5-30%改善、コスト10-20%削減はよく出る数値レンジ
- 出典: マイナビ転職／doda／JAC Recruitment

### Block 3: ベンダーコントロール経験を「目的→調整→成果」で書く
- ポイント: 折衝力ワードだけ書いても評価されない
- 詳細:
  - JAC「『調整力』『折衝力』とワードだけでは評価されない。何のために調整したのか、課題→目的→成果のストーリーを書け」
  - 必須要素: ベンダー社数・契約金額・期間・自分の役割（PM/発注側担当/PMO等）
  - 例: 「3社相見積（合計1.2億）→ 要件定義同席で仕様変更コスト**400万削減**／納期**2週短縮**」
  - 社内SE求人の頻出キーワード「ベンダーコントロール」は単独求人が少なく**社内SE職務の一部**として評価される（リクルートエージェント／doda求人）
- 出典: JAC Recruitment／doda求人／techmania

### Block 4: 客先名は伏せ、ステークホルダー数で代替する（SES特化）
- ポイント: 守秘義務を守りつつ「動かした人数」で示す
- 詳細:
  - 客先企業名は「大手通信会社（売上3兆円規模）」のように**抽象化**（社内SEナビ）
  - 代わりに**動かした人数**を出す: 利用部門ユーザー数・関係部署数・経営層プレゼン回数
  - SESで陥りがちな失敗: 体制図上の自分の位置（プロパー/協力会社）が不明、誰の指示で動いたかが曖昧
  - 体制図に「★私」を入れて発注側/受注側どちらの立場かを明示するだけで読みやすさが変わる
- 出典: 社内SEナビ／GeeklyMedia／r35-se（採用担当歴5年）

### Block 5: 「事業会社の言葉」に翻訳する（最重要）
- ポイント: 顧客対応 → 合意形成、納期管理 → リスク管理
- 詳細:
  - SIer語 → 事業会社語の翻訳辞書:
    - 「顧客との仕様調整」→「複数ステークホルダー間の合意形成」
    - 「テスト消化」→「品質保証プロセスの設計と運用」
    - 「ベンダー管理」→「外部リソース活用による内製コスト最適化」
    - 「保守運用」→「事業継続性の担保とインシデント対応」
  - 評価軸はSIerと真逆: 技術＞人柄ではなく**人柄≧技術**、新技術導入より**既存業務の改善**、スピードより**正確性**
- 出典: えんじこーむnote（SIer→事業会社の言語翻訳論）／既存メモ interview_question_patterns.md

### Block 6: コスト・セキュリティ・ガバナンス視点を入れる
- ポイント: 社内SE特有の3視点で1段差をつける
- 詳細:
  - 社内SEは「自社のコストと業務効率を意識」が必須評価軸（リクルートエージェント）
  - コスト: ライセンス見直しで年間**X円削減**／クラウド最適化で月**Y%減**
  - セキュリティ: ISMS運用、ゼロトラスト導入、Intune/Jamf 端末管理台数
  - ガバナンス: シャドーIT棚卸し、SaaS数（参考: 1,000名以上企業の平均207サービス／IPA）
  - これを書ける応募者は実は少ない（SIer/SESではコスト責任を持たないため）→ **差別化箇所**
- 出典: リクルートエージェント／既存メモ trending_facts_2026_05.md／IPA

### Block 7: 3分で読める構成にする（読みやすさ＝評価）
- ポイント: 採用担当は1人あたり3分しか読まない
- 詳細:
  - 推奨ボリューム: **A4で2〜3枚**（多くて4枚）／冒頭に**職務要約5-7行**で全体像
  - 箇条書きを多用しリズムを作る（マイナビ／doda／日経転職版で共通推奨）
  - フォーマット崩れ・誤字脱字は「大切な書類1つ正確に作れない人」と判定（r35-se 採用担当歴5年）
  - 推奨構成: ①職務要約 → ②会社概要 → ③プロジェクト経歴（時系列・降順） → ④テクニカルスキル一覧 → ⑤資格 → ⑥自己PR
- 出典: マイナビ転職／doda／日経転職版／r35-se

## キーメッセージ（図解の周辺テキスト用）
- 通過率37.3%は「書き方」で動く（25字）
- 動詞を「成果動詞＋数字」に置換（21字）
- 客先名は伏せ、人数で代替する（18字）
- SIer語→事業会社語に翻訳する（17字）
- ジョブ型でJDマッチ度が見られる（19字）

## 落とし穴・注意点
- **通過率37.3%は全職種平均**：社内SEは応募集中職のため実態は更に低いが、職種別の公表値は存在しない（doda/マイナビとも非公開）
- **「ストーリー性で2倍」の出典**：人材紹介CRM業者LaSの記事に登場するが**1次データなし**。本MDでは採用しない
- **添削サービスの通過率改善エビデンス**：「平均30%台に上がる」と書く記事はあるが、対照群を取った定量研究は見つからず（=「上がる傾向」までしか言えない）
- **誇張・盛りすぎ厳禁**：面接で深掘りされて答えられない数字は逆効果（社内SEナビ）
- **求人票キーワードのコピペ**：JD要件のキーワードを並べるだけだとAI生成と疑われる。**自分の動詞**で書き換える

## 次のアクション（CTA候補）
1. 今の職務経歴書を**7項目で自己採点**（各2点／14点満点／10点未満は要書き換え）
2. 「対応した／担当した」を**Ctrl+F**して全部書き換える（5分でできる）
3. 数字が3個以下なら**3個以上に増やす**（規模／成果／コスト）
4. 客先名・社内固有名詞を**抽象化＋人数化**する
5. 自分でやり切れない／2社目以降も落ち続ける場合 → **社内SE特化エージェントの添削**を受ける（無料・対面で改善点を聞ける）

## 研究メモ（図解には載せないが diagrammer/reviewer が参考にする情報）

### 確定数値の出典まとめ
- **マイナビ2025**: 通過率37.3%／応募13.6件／通過5.1件／有効回答467名（2025年7月10-15日調査）
- **doda**: 個別利用ベース22%、転職成功者の平均応募32社（2024年1-12月実績）
- **マイナビ転職**: 一般通過率30〜50%（出典明示なしだが各社共通レンジ）
- **レバテック**: IT求人倍率10.4倍（2025/12）、セキュリティ42.6倍

### ジョブ型雇用×書類選考の構造変化
- 2024/8/28 内閣官房・経産省・厚労省「ジョブ型人事指針」公表（先進20社事例集）
- 日立: 全職種・全等級で**450種**の標準JD作成、全ポジションのJD開示
- 富士通: 2020年管理職15,000名 → 2022年非管理職45,000名 → **2026年4月新卒からジョブ型全面移行**
- 影響: 求人票の「必須要件」「歓迎要件」がJD化され、**書類段階でスキル要件マッチを機械的にチェック**される傾向が強まる

### 社内SE特有の評価軸（複数ソース合意）
1. **業界・システム規模・役割**（dodaチェックリスト最重要項目）
2. **ベンダーコントロール経験**（JAC、社内SEナビ）
3. **業務理解・利用部門折衝**（リクルート、日経）
4. **コスト意識・経営視点**（リクルートエージェント）
5. **継続性・再現性・カルチャーフィット**（採用担当の暗黙評価軸／既存メモ参照）

### SIer/SES出身者が陥りがちな失敗パターン（複数ソース集約）
- 抽象表現の多用（「対応」「担当」「貢献」）
- 客先企業名が伏せられず守秘違反、または完全空白で経歴が見えない
- 体制図で自分の位置（発注側／受注側／プロパー／協力会社）が不明
- 技術スタック羅列のみで「使って何を解決したか」が不在
- ネガティブ志望動機（残業／客先常駐に疲れた等）→ 即落ち
- 35歳超でマネジメント経験記載なし → 自動落ち（r35-se）
- 募集要項を読まずテンプレ使い回し（=「ルール無視」と判定）

### 添削サービス活用の根拠（控えめに採用）
- 「通過率は平均30%台まで上がる」（複数エージェント記事の主張、対照群なし）
- マイナビ転職エージェント等の主張: 過去成功例の構成・言い回しを直接共有
- 客観エビデンスは弱いが、**書類段階で第三者レビューを入れる効果**は職務経歴書添削サービス各社が共通主張
- アフィリエイト導線: 「自分で2-3社目以降も落ち続けたら専門エージェントの添削が転換点」と自然接続

### 1枚図解のフレーミング案
- A案（採用）: 7項目チェックリスト＋上部に通過率37.3%／下部にCTA
- B案: Before/After 書き換え例（「対応した」→「200件→90件削減」）4対比
- C案: SIer語/事業会社語 翻訳辞書

A案を主推奨：保存価値が最も高い（読者が職務経歴書を書くたびに見返す永久保存版として機能）。B/Cはサブ素材として研究メモに保持。

### 数字の鮮度
- 全数値は2024-2026年公表分。1年経過後の使用前は更新確認必須
- 特にマイナビ調査は毎年9月頃更新／doda求人倍率は四半期更新
