---
name: 転職サービス固有名詞での断罪は避ける
description: 「使ってはいけない転職サイト」系トピックで固有名詞を出さず「タイプ・類型」に抽象化する運用ルール
type: feedback
---

転職サービス・エージェントに関する記事で、特定サービスを名指しで「使ってはいけない」「ダメ」と断定しない。代わりに**サービスの“タイプ／類型”**として一般化する。

**Why:**
- 景品表示法・名誉毀損リスク（事実の摘示でも社会的評価低下なら成立し得る）
- アフィリエイトASP（A8/もしも/afb等）の規約違反リスク
- 同じ運営会社でもブランドにより性格が違うため、固有名詞での一律評価そのものが不正確
- 担当者個人の質に体験が依存するため「サービス全体がダメ」と即断するのは事実関係としても弱い

**How to apply:**
- トピックが「○○系サービスの問題点」を扱う場合、固有名詞ではなく**①特徴 → ②なぜ不向きか → ③見抜き方 → ④代替**の構造で類型化
- 具体例を出すなら「○○のような特徴を持つサービス」レベルの一般化に留める
- 落とし穴セクションで「同じ運営会社でもブランドにより性格が違う」「体験は人による」を明記し、読者の早合点をブロック
- 数字（求人倍率、応募数など）は二次情報でも複数ソースで一致する範囲のみ採用、出典を併記
