社内SE書類選考通過率と職務経歴書評価観点の即引きファクト集（2026年5月時点）。

## 通過率の1次数値（出典明示）
- マイナビ「転職活動実態調査（2025年）」: **書類選考通過率37.3%／平均応募13.6件／通過5.1件**（有効回答467名・2025年7月10-15日調査）
- doda: 個別利用ベース約22%、転職成功者平均応募32社・面接通過5社（2024年1-12月実績）
- マイナビ転職一般水準: **30〜50%**（出典明示なし共通レンジ）
- IT求人倍率: レバテック10.4倍（2025/12）／セキュリティ42.6倍
- 社内SEは1ポスト100-200倍応募の人気職（doda入社決定職種で最多）

## 採用担当が見る評価軸（複数ソース合意）
1. 業界・システム規模・役割（dodaチェックリスト最重要）
2. ベンダーコントロール経験（JAC・社内SEナビ）
3. 業務理解・利用部門折衝（リクルート・日経）
4. コスト意識・経営視点（リクルートエージェント）
5. 継続性・再現性・カルチャーフィット（暗黙軸）

## SIer/SES出身者の頻出失敗パターン
- 抽象動詞「対応した／担当した／貢献した」の多用
- 客先企業名の扱い（伏せ過ぎor守秘違反）
- 体制図で自分の位置（発注/受注/プロパー/協力）が不明
- 技術スタック羅列のみで「使って何を解決したか」不在
- ネガティブ志望動機（残業／客先常駐疲労）→ 即落ち
- 35歳超でマネジメント経験記載なし → 自動落ち（r35-se 採用担当歴5年）
- 募集要項読まずテンプレ使い回し → 「ルール無視」判定

## SIer語→事業会社語の翻訳辞書
- 顧客との仕様調整 → 複数ステークホルダー間の合意形成
- テスト消化 → 品質保証プロセスの設計と運用
- ベンダー管理 → 外部リソース活用による内製コスト最適化
- 保守運用 → 事業継続性の担保とインシデント対応

## ジョブ型雇用の構造変化（書類選考への影響）
- 2024/8/28 内閣官房・経産省・厚労省「ジョブ型人事指針」公表（先進20社事例）
- 日立: 全職種全等級で**450種**の標準JD作成・全ポジション開示
- 富士通: 2020年管理職15,000名→2022年非管理職45,000名→2026年4月新卒からジョブ型全面移行
- 影響: 求人票JD化で書類段階のスキル要件マッチが機械的判定に近づく

## How to apply
- 通過率を語る時はマイナビ37.3%を主、doda22%・一般30-50%を副で併記
- 添削サービスCTAは「2-3社落ちたら添削が転換点」のフレームで自然接続
- 「動詞を成果動詞＋数字に置換」は書類選考系記事の核フック
- 体制図で「★私」の位置を明示するTipsは保存率高い見込み
