name: CONTENT_DATAカードの命名と構成原則
description: 9:16図解の7枚カード構成で読みやすさと保存価値を両立する書き方
type: feedback
---

**鉄則**: チェックリスト型テーマは1カード=1動作。見出しは「動詞or名詞句10字以内」、説明文は「具体例＋数字」、補足ラベルは「立ち位置/分類8字以内」。

**Why:**
- 9:16画像は1画面の情報密度が低い（スマホで一瞥）ため、見出しが抽象的だと中身が伝わらない
- 補足ラベル（「最重要」「SES特化」「差別化」等）が並ぶと、視覚的に読者の現状と該当カードがマッチング判定しやすくなる
- ストーリー順（規模感→動詞置換→ベンダー→人数化→翻訳→コスト→構成）は浅い→深いの設計にすると、保存価値が上がる

**How to apply:**
- カード見出しは命令形or体言止めで揃える（混在しない）
- 説明文には必ず具体例 or 比較（「○○→○○」「例: ○○」）を入れる
- 補足ラベルは「最重要」「差別化」「数字で示す」など、読者がスキャン時に飛ばすか読むかの判断材料になる短語を選ぶ
- 数字は7枚中1〜2枚に絞り、最も訴求力ある箇所（フック直後 or 中盤）に集中配置
- 全角縦棒「｜」厳守、半角「|」は画像生成スクリプトでパース失敗する
